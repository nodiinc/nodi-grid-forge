# Grid Forge — 제품 소개서

AI 기반 차세대 SCADA 화면 생성 플랫폼

---

## 1. 기존 SCADA의 문제

### 화면 제작이 느리고 비싸다

기존 SCADA 화면 제작은 **전문 엔지니어가 에디터에서 수작업**으로 수행한다.

```text
기존 프로세스:
  1. 현장 조사 및 요구사항 수집         1~2일
  2. P&ID 기반 화면 설계               1~2일
  3. SCADA 에디터에서 도형 배치         2~5일
  4. 태그 연결, 애니메이션 설정         1~3일
  5. 알람/제어 설정                    1~2일
  6. 테스트 및 수정                    1~2일
  ──────────────────────────────────
  화면 1개당 총 소요: 7~16일 (전문 엔지니어 기준)
```

### 변경 비용이 높다

현장 설비가 변경되거나, 운영자가 화면 레이아웃을 바꾸고 싶을 때:

- 기존: 엔지니어에게 다시 의뢰 → 수일~수주 소요
- 긴급 변경: 엔지니어 일정 확보 → 현장 방문 → 수정 → 검증
- 비용: 변경 건당 수십~수백만 원 (외주 기준)

### 엔지니어 의존도가 높다

- SCADA 화면 제작/수정에 전문 교육이 필요 (수개월 학습)
- 엔지니어 퇴사 시 유지보수 리스크
- 현장 운영자는 화면 수정 불가 → 불편을 감수하며 사용

## 2. Grid Forge의 해결 방식

### 자연어와 이미지로 SCADA 화면 생성

```text
Grid Forge 프로세스:
  1. 사용자: "수배전반 화면 만들어줘" + 스케치 이미지 첨부
  2. AI: 컴포넌트 조립 + 코드 생성 → 실시간 Preview 표시   수분
  3. 사용자: "차단기에 투입/개방 버튼 추가해줘"
  4. AI: 코드 수정 → 즉시 반영                             수초
  5. 반복 수정 (자연어로 계속 대화)
  ──────────────────────────────────────────────────
  화면 1개당 총 소요: 수분~수십분
```

### 핵심 기술: 컴포넌트 조립 + AI 바이브코딩

검증된 SCADA 컴포넌트 라이브러리를 AI가 **조립**하고, 특수한 부분만 **직접 생성**한다.

```text
AI가 하는 일:                     컴포넌트가 하는 일:
  레이아웃 결정                      장비 심볼 렌더링 (SVG)
  컴포넌트 배치 + 태그 바인딩         상태 색상/애니메이션
  커스텀 요소 직접 코드 생성          측정값 표시 (포맷/경보)
  → 50~100줄 생성                   연결선/모선 렌더링
                                   제어 확인 패턴
                                   → 항상 동일한 품질
```

- **품질 일관성**: 차단기, 변압기, 모선 등 핵심 심볼은 항상 동일한 품질로 렌더링
- **대규모 화면 대응**: 장비 30개 이상도 컴포넌트 배치만으로 구성 가능
- **토큰 효율**: AI 출력 70% 절감 (300줄 → 80줄), 비용 $0.07 → $0.02/회
- **유연성 유지**: 컴포넌트에 없는 현장 고유 요소는 AI가 직접 생성

### 다양한 화면 수정 방식

```text
방법 1: 텍스트
  "전압 게이지 크기를 키우고, 과전류 시 빨간색 깜박이게 해줘"

방법 2: 이미지 + 텍스트
  현재 화면 스크린샷에 빨간 동그라미 표시 후
  "여기에 트렌드 차트 추가해줘"

방법 3: 기존 SCADA 캡처 → AI 재생성
  기존 시스템 화면 스크린샷 첨부 후
  "이 화면을 Grid Forge에서 동일하게 만들어줘"

방법 4: 인앱 드로잉 (Phase 3)
  화면 위에 직접 화살표, 메모, 영역 표시 → AI가 이해하고 수정
```

### 대화 이력 관리

AI와의 대화 기록은 자동으로 보존된다.

- 이전 대화를 이어서 수정 작업 계속 가능
- 대화 초기화(Clear)해도 생성된 화면은 그대로 유지
- 새 대화에서도 기존 화면 코드를 기반으로 수정 가능
- 대화 이력은 화면별로 격리 (다른 화면의 대화가 섞이지 않음)

```text

    향후: 모델 티어링, 템플릿 활용으로 추가 절감
```

## 4. 기능 상세

### 4.1 실시간 데이터 연동

Edge Gateway를 통해 현장 설비와 실시간으로 통신한다. **기존 설비와 게이트웨이는 변경 없이 그대로 사용한다.**

```text
지원 프로토콜:
  Modbus TCP/RTU    — PLC, 계전기, 전력량계
  OPC UA            — 산업 표준 통합 프로토콜
  IEC 61850         — 디지털 변전소
  MQTT              — IoT 센서, 클라우드 연동
  SNMP              — 네트워크 장비 모니터링
  REST/HTTP         — 외부 시스템 API 연동

데이터 흐름:
  [전력 설비] ←→ [Edge Gateway] ←→ [TagBus] ←→ [Grid Forge 서버] ←→ [브라우저]
                                                                      │
                                              실시간 태그값 표시 + 제어 명령
```

### 4.2 알람/이벤트/로그 관리

설비 이상 상태를 자동으로 감지하고, 운전원에게 즉시 알린다. 정형화된 통합 관리 화면을 제공한다.

```text
알람 워크플로우:
  1. 알람 규칙 정의    — "TR1 전류 > 400A이면 과전류 경보"
  2. 실시간 평가       — Data Worker가 태그값을 조건과 비교
  3. 알람 발생         — 화면 배너 표시, 색상 변경, 이력 기록
  4. 운전원 확인       — Acknowledge 버튼으로 인지 확인
  5. 조건 해소         — 정상 복귀 시 자동 해제, 이력 보존

통합 관리 화면 (글로벌 네비게이션에서 접근):
  - [실시간 알람]: 활성 알람 실시간 표시, 개별/전체 Acknowledge
  - [알람 이력]:   기간/심각도/장비 필터링, 통계 요약, CSV/PDF 내보내기
  - [이벤트 로그]: 장비 상태 변경, 제어 명령 이력
  - [시스템 로그]: 로그인/설정변경/AI 생성 이력

운영 안정성:
  - 알람 억제/쉘빙: 유지보수 중 불필요 알람 일시 비활성
  - 알람 홍수 방지: 그룹핑, 지연, 우선순위 큐
  - AI 연동: 화면 생성 시 알람 표시 자동 포함
```

### 4.3 설비 제어

운전원이 화면에서 직접 설비를 제어할 수 있다. nodi-edge를 통한 자동 제어도 지원한다.

```text
수동 제어 (화면 UI):
  [운전원 버튼 클릭] → [확인 다이얼로그] → [서버 권한 검증]
  → [속도 제한 체크] → [감사 로그 기록] → [Gateway → 설비 제어]

자동 제어 (nodi-edge 스크립트):
  Edge Gateway 내부 스크립트 앱이 조건 기반 자동 제어 수행
  예: 변압기 부하율 > 95% → 우선순위 낮은 부하 자동 차단

안전장치:
  1. 2단계 확인 패턴    — 위험 제어는 반드시 2회 클릭 확인
  2. 역할 기반 제한     — Admin/Operator만 제어 가능, Viewer는 비활성
  3. 속도 제한          — 초당 제어 명령 횟수 제한 (오조작 방지)
  4. 인터록             — 위험 조건에서 제어 명령 자동 거부
  5. 감사 로그          — 누가, 언제, 어떤 제어를 했는지 전체 기록
```

### 4.4 이력 데이터 및 트렌드

시계열 데이터의 수집/저장/애그리게이션은 Edge Gateway가 수행하고, Grid Forge는 조회/시각화를 담당한다.

```text
정형화된 이력 조회 화면:
  - 태그 선택 (장비 트리 / 검색)
  - 기간/해상도/집계 방식 설정
  - 트렌드 차트 (다축, 줌/팬, 알람 영역 표시)
  - 데이터 테이블 (정렬, 통계 요약, CSV 내보내기)

AI 연동:
  - AI가 화면 내에 TrendChart 컴포넌트를 직접 배치
  - "변압기 전류 추이를 화면 하단에 표시해줘" → 즉시 반영
```

### 4.5 리포트

운영 데이터를 기반으로 정기/수시 리포트를 자동 생성한다.

```text
리포트 유형:
  - 정기: 일보, 월보, 연보 (자동 생성)
  - 수시: 사고 분석, 구간 분석, 비교 분석
  - 규정 준수: 전력거래소/환경부 제출용

출력/배포:
  - PDF / Excel 다운로드
  - 이메일 자동 발송 (구독 설정)
  - 대시보드 내 리포트 요약 임베드
```

### 4.6 화면 버전 관리

모든 화면 변경은 버전으로 관리되며, 즉시 롤백할 수 있다.

```text
버전 관리:
  v1  "수배전반 단선도 생성"          2024-03-08 14:30  김운영
  v2  "TR-3 변압기 추가"             2024-03-08 15:10  김운영
  v3  "차단기 색상 변경"              2024-03-08 15:25  김운영  ← 현재

  v3에 문제 발견 → v2로 원클릭 롤백 → 즉시 이전 화면 복원

  - 변경 이력 전체 추적 (누가, 언제, 무엇을 변경했는지)
  - 이전 버전과 현재 버전 코드 비교 가능
  - 롤백 시 기존 화면이 즉시 복원 (컴파일 대기 없음)
```

### 운영 이점

- **엔지니어 의존도 제거**: 현장 운영자가 자연어로 화면 수정
- **즉각적 변경 대응**: 현장 변경 → 즉시 화면 반영 (대기 시간 없음)
- **표준화된 품질**: 컴포넌트 라이브러리 + Knowledge 시스템으로 일관된 화면 품질
- **안전한 배포**: AST 검증 + 관리자 승인 워크플로우
- **완전한 감사 추적**: 모든 화면 변경, 제어 명령, 알람 확인을 기록
- **즉시 롤백**: 문제 발생 시 이전 버전으로 원클릭 복원

## 6. 보안 및 운영 안정성

### 다계층 보안 (Defense in Depth)

```text
Layer 1: AI 도구 제한       — 사용 가능한 도구를 5개로 제한 (파일시스템/쉘 접근 불가)
Layer 2: AST 코드 검증      — AI 생성 코드에서 금지 패턴 자동 검출
Layer 3: 컴파일 격리         — esbuild에서 외부 모듈 차단 (네트워크 접근 불가)
Layer 4: 런타임 권한         — 서버사이드 권한/속도 제한 강제
Layer 5: 배포 승인           — 관리자 승인 + 감사 로그
```

### 운영 안정성

- **24/7 가동**: 장애 발생 시 자동 복구 (systemd, 재접속, 스냅샷 복원)
- **Graceful Degradation**: AI 장애 시에도 실시간 모니터링/제어/알람은 정상 동작
- **버전 롤백**: 원하지 않는 변경 발생 시 이전 버전으로 즉시 복원
- **완전한 감사 추적**: 화면 변경, 제어 명령, 알람 확인 전체 기록

### 운영 거버넌스

```text
역할별 권한:
  Viewer     — 화면 조회만 가능
  Operator   — 모니터링 + 설비 제어 (승인된 화면에서)
  Engineer   — AI를 통한 화면 생성/수정 + Preview 확인
  Admin      — 화면 배포 승인 + 사용자/역할 관리

워크플로우:
  화면 생성/수정     → Engineer 이상 (AI를 통해 수행)
  동작 확인          → 사용자가 Preview에서 검증
  배포 승인          → Admin 이상 (검증 통과 필수)
  운영 모니터링/제어  → Operator 이상
  문제 발생 시       → 즉시 롤백 또는 배포 회수

  AI가 도구를 제공하고, 운영 판단과 배포 책임은 고객사가 수행합니다.
```

## 8. AI 엔진 — 상용 API vs 온프레미스 LLM

### 왜 상용 API인가

Grid Forge의 AI 엔진은 SCADA 화면 코드를 직접 생성한다. 이는 단순한 텍스트 생성이 아니라, React 코드 생성 + SVG 좌표 계산 + 도구 호출을 정확히 수행해야 하는 고난도 작업이다. 이 모든 능력을 **동시에 높은 수준으로** 갖춘 모델은 현재 상용 API뿐이다.

### 상용 API vs 온프레미스 비교

```text
                    상용 API              온프레미스 70B급        온프레미스 600B+급
                    (Claude Sonnet)       (Llama/Qwen 70B)       (DeepSeek-V3 등)
─────────────────────────────────────────────────────────────────────────────────
코드 생성 품질
  100줄 이하         매우 좋음             좋음                   좋음
  300줄+ 복잡한 화면  매우 좋음             낮음~보통               보통
  기존 코드 수정      매우 좋음             낮음                   보통

도구 호출 정확도      매우 높음             불안정                  보통

초기 비용            $0                   $15,000+ (GPU 1장)     $60,000+ (GPU 4장)
월 운영비 (500회)    ~$10                 $60~1,500              $120+
응답 속도            빠름 (수초)           보통 (수초~수십초)       느림 (수십초)
모델 업데이트        자동 (최신 모델 즉시)   수동 (다운로드+배포)     수동
가용성              99.9%+ (SLA)         자체 관리               자체 관리
```

### 비용 분석

```text
시나리오: 월 500회 화면 생성/수정

[상용 API]
  초기 비용: $0
  월 비용:   ~$10 (컴포넌트 기반 + 토큰 최적화)
  연간:      ~$120

[온프레미스 70B급]
  초기 비용: $15,000+ (GPU 서버)
  월 비용:   ~$60 (전력 + 유지보수)
  연간:      $15,000 + $720

결론: 월 수천 회 미만이면 API가 항상 경제적이다.
```

### 온프레미스가 필요한 경우

보안 규제로 태그 정보/화면 코드가 외부 API로 나갈 수 없는 환경(발전소, 군사시설, 폐쇄망)에서만 온프레미스가 필요하다. 이 경우에도 코드 품질 저하, GPU 인프라 비용, 운영 인력이 추가로 필요하다.

### Grid Forge의 전략

```text
현재: 상용 API (Claude Sonnet) 사용
  → 최고 품질, 즉시 시작, 비용 최소

아키텍처: AI 엔진을 추상화하여 LLM 교체 가능하게 설계
  → 오픈소스 모델 품질 향상 시 온프레미스 옵션 제공
  → 고객사 보안 요건에 따라 API/온프레미스 선택 가능

비용 최적화 (적용 완료):
  1. 컴포넌트 기반     — AI 출력 토큰 70% 절감
  2. Prompt Caching   — 시스템 프롬프트 반복 비용 72% 절감
  3. 히스토리 압축     — 수정 요청 시 이전 대화 50% 절감
```

## 9. 도입 시나리오

### PoC (2~4주)

```text
목표: AI SCADA 화면 생성의 실현 가능성 확인

범위:
  - 단일 현장, 실제 Edge Gateway 연동
  - 3~5개 대표 화면 AI 생성
  - 실시간 태그 표시 + 기본 알람
  - 이미지 기반 화면 수정 체험

결과물:
  - 실제 동작하는 SCADA 화면 (AI 생성)
  - 기존 대비 제작 시간 비교 데이터
  - 현장 운영자 사용성 피드백
```

### 파일럿 (1~3개월)

```text
목표: 실제 운영 환경에서 안정성 검증

범위:
  - 단일 현장 전체 화면 (10~20개)
  - 제어 기능 포함 (2단계 확인 + 감사 로그)
  - 알람 시스템 전체 운영 (발생 → 확인 → 해제)
  - 화면 버전 관리 + 롤백 운영
  - 운영자 교육 + 직접 화면 수정 체험

결과물:
  - 운영 안정성 보고서 (가동률, 장애 건수)
  - 운영자 자립도 평가 (교육 후 독립 운영 가능성)
  - 전사 확대 ROI 산출
```

```text
기대 효과:
  - 화면 제작 비용 90%+ 절감
```
