# Grid Forge — 제품 소개서

AI 기반 차세대 SCADA 화면 생성 플랫폼

---

## 1. 기존 SCADA의 문제

### 화면 제작이 느리고 비싸다

기존 SCADA 화면 제작은 **전문 엔지니어가 에디터에서 수작업**으로 수행한다.

```text
기존 프로세스:
  1. 현장 조사 및 요구사항 수집         1~2일
  2. P&ID 기반 화면 설계               1~2일
  3. SCADA 에디터에서 도형 배치         2~5일
  4. 태그 연결, 애니메이션 설정         1~3일
  5. 알람/제어 설정                    1~2일
  6. 테스트 및 수정                    1~2일
  ──────────────────────────────────
  화면 1개당 총 소요: 7~16일 (전문 엔지니어 기준)
```

### 변경 비용이 높다

현장 설비가 변경되거나, 운영자가 화면 레이아웃을 바꾸고 싶을 때:

- 기존: 엔지니어에게 다시 의뢰 → 수일~수주 소요
- 긴급 변경: 엔지니어 일정 확보 → 현장 방문 → 수정 → 검증
- 비용: 변경 건당 수십~수백만 원 (외주 기준)

### 엔지니어 의존도가 높다

- SCADA 화면 제작/수정에 전문 교육이 필요 (수개월 학습)
- 엔지니어 퇴사 시 유지보수 리스크
- 현장 운영자는 화면 수정 불가 → 불편을 감수하며 사용

## 2. Grid Forge의 해결 방식

### 자연어와 이미지로 SCADA 화면 생성

```text
Grid Forge 프로세스:
  1. 사용자: "수배전반 화면 만들어줘" + 스케치 이미지 첨부
  2. AI: React 코드 생성 → 실시간 Preview 표시           수분
  3. 사용자: "차단기에 투입/개방 버튼 추가해줘"
  4. AI: 코드 수정 → 즉시 반영                           수초
  5. 반복 수정 (자연어로 계속 대화)
  ──────────────────────────────────────────────────
  화면 1개당 총 소요: 수분~수십분
```

### 핵심 기술: AI 바이브코딩

AI가 미리 정해진 컴포넌트를 조합하는 것이 아니라, **React 코드를 직접 생성**한다.

- 사전 컴포넌트 방식의 한계: 모든 현장의 요구를 예측 불가
- 바이브코딩: 현장별 고유한 레이아웃, 장비 구성, 표현 방식을 자유롭게 구현
- AI가 SCADA 도메인 지식을 학습하여 전문 엔지니어 수준의 화면 생성

### 직관적 수정 방식

```text
방법 1: 텍스트
  "전압 게이지 크기를 키우고, 과전류 시 빨간색 깜박이게 해줘"

방법 2: 이미지 + 텍스트
  현재 화면 스크린샷에 빨간 동그라미 표시 후
  "여기에 트렌드 차트 추가해줘"

방법 3: 인앱 드로잉 (Phase 3)
  화면 위에 직접 화살표, 메모, 영역 표시 → AI가 이해하고 수정
```

## 4. 기존 SCADA 대비 도입 이점

### 기능 비교

```text
항목                     기존 SCADA         Grid Forge
───────────────────────────────────────────────────────────
화면 제작 방식            수작업 에디터       자연어 + AI
제작 소요 시간            수일~수주           수분~수십분
수정 방식                에디터 재작업        대화로 즉시 수정
필요 전문성               SCADA 에디터 교육   자연어 (한국어)
실시간 데이터 연동         ✓                 ✓
설비 제어                 ✓                 ✓
알람 시스템               ✓                 ✓
시계열 트렌드             ✓                 ✓
이미지 기반 수정           ✗                 ✓ (스케치/스크린샷)
인앱 드로잉 어노테이션      ✗                 ✓ (Phase 3)
AI 기반 레이아웃 제안       ✗                 ✓
버전 관리 + 롤백           제한적             ✓ (전체 이력 + 즉시 롤백)
다중 사이트 관리           별도 구매           기본 제공 (멀티테넌시)
```

### 운영 이점

- **엔지니어 의존도 제거**: 현장 운영자가 자연어로 화면 수정
- **즉각적 변경 대응**: 현장 변경 → 즉시 화면 반영 (대기 시간 없음)
- **표준화된 품질**: AI + Knowledge 시스템으로 일관된 화면 품질 유지
- **안전한 배포**: AST 검증 + 관리자 승인 워크플로우
- **완전한 감사 추적**: 모든 화면 변경, 제어 명령, 알람 확인을 기록

## 7. 보안 및 운영 안정성

### 다계층 보안 (Defense in Depth)

- **Layer 1**: AI가 사용할 수 있는 도구를 5개로 제한 (파일시스템/쉘 접근 불가)
- **Layer 2**: AI 생성 코드를 AST 분석으로 금지 패턴 검출
- **Layer 3**: 컴파일 단계에서 외부 모듈 차단
- **Layer 4**: 런타임에서 서버사이드 권한/속도 제한 강제
- **Layer 5**: 관리자 배포 승인 + 감사 로그

### 운영 안정성

- **24/7 가동**: 장애 발생 시 자동 복구 (systemd, 재접속, 스냅샷 복원)
- **Graceful Degradation**: AI 장애 시에도 실시간 모니터링/제어/알람은 정상 동작
- **버전 롤백**: 원하지 않는 변경 발생 시 이전 버전으로 즉시 복원
- **완전한 감사 추적**: 화면 변경, 제어 명령, 알람 확인 전체 기록

### 운영 거버넌스

```text
  화면 생성/수정     → Engineer 이상 (AI를 통해 수행)
  동작 확인          → 사용자가 Preview에서 검증
  배포 승인          → Admin 이상 (컴파일 + 검증 통과 필수)
  운영 모니터링/제어  → Operator 이상
  문제 발생 시       → 즉시 롤백 또는 배포 회수

  AI가 도구를 제공하고, 운영 판단과 배포 책임은 고객사가 수행합니다.
```
